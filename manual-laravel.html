¿Qué es Laravel?

Framework PHP MVC de alto nivel

Laravel es el framework PHP más popular del mundo. Sigue el patrón MVC (Modelo-Vista-Controlador) y provee herramientas para construir aplicaciones web y APIs de manera elegante, rápida y segura.

Características principales

  • Eloquent ORM (base de datos)
  • Blade (motor de plantillas)
  • Artisan (CLI poderosa)
  • Migraciones de base de datos
  • Sistema de colas y jobs
  • Broadcasting en tiempo real
  • Sanctum & Passport (auth)
  • Testing integrado con Pest

Versión actual: Laravel 12

  • Lanzado: 24 febrero 2025
  • PHP mínimo: 8.2
  • Basado en Symfony 7
  • Soporte bugs: 18 meses
  • Soporte seguridad: 2 años
  • Starter kits: React, Vue, Svelte, Livewire
  • Inertia.js v2 + shadcn/ui
  • WorkOS AuthKit (SSO, passkeys)

Configuración de Apache desde cero

Paso 1 — Instalar Apache y PHP

sudo apt update
sudo apt install apache2 php8.2 libapache2-mod-php8.2 php8.2-mysql \
  php8.2-mbstring php8.2-xml php8.2-curl php8.2-zip php8.2-bcmath

# Habilitar mod_rewrite (OBLIGATORIO para Laravel)
sudo a2enmod rewrite
sudo systemctl restart apache2

Paso 2 — Colocar el proyecto

sudo cp -r mi-proyecto /var/www/mi-proyecto

# Ajustar permisos (MUY IMPORTANTE)
sudo chown -R www-data:www-data /var/www/mi-proyecto
sudo chmod -R 755 /var/www/mi-proyecto
sudo chmod -R 775 /var/www/mi-proyecto/storage
sudo chmod -R 775 /var/www/mi-proyecto/bootstrap/cache

Paso 3 — Crear VirtualHost

sudo nano /etc/apache2/sites-available/mi-proyecto.conf
<VirtualHost *:80>
    ServerName miapp.local
    ServerAlias www.miapp.local
    DocumentRoot /var/www/mi-proyecto/public

    <Directory /var/www/mi-proyecto/public>
        AllowOverride All
        Require all granted
        Options -Indexes +FollowSymLinks
    </Directory>

    ErrorLog ${APACHE_LOG_DIR}/laravel-error.log
    CustomLog ${APACHE_LOG_DIR}/laravel-access.log combined
</VirtualHost>

Paso 4 — Activar el sitio

sudo a2ensite mi-proyecto.conf
sudo a2dissite 000-default.conf
sudo systemctl reload apache2

# Para desarrollo local:
echo "127.0.0.1 miapp.local" | sudo tee -a /etc/hosts

Paso 5 — .htaccess (ya incluido en Laravel)

<IfModule mod_rewrite.c>
    <IfModule mod_negotiation.c>
        Options -MultiViews -Indexes
    </IfModule>

    RewriteEngine On
    RewriteCond %{HTTP:Authorization} .
    RewriteRule .* - [E=HTTP_AUTHORIZATION:%{HTTP:Authorization}]

    RewriteCond %{REQUEST_FILENAME} !-d
    RewriteCond %{REQUEST_FILENAME} !-f
    RewriteRule ^ index.php [L]
</IfModule>
El DocumentRoot debe apuntar a la carpeta /public, nunca a la raíz del proyecto. Esto protege archivos sensibles como .env.

Configuración de CORS
CORS (Cross-Origin Resource Sharing) permite o bloquea peticiones desde dominios distintos a tu API. En Laravel se configura en config/cors.php.

config/cors.php completo

return [
    // Rutas que aplican CORS
    'paths' => ['api/*', 'sanctum/csrf-cookie'],

    // Métodos HTTP permitidos
    'allowed_methods' => ['*'],

    // Orígenes permitidos
    'allowed_origins' => [
        'https://mi-frontend.com',
        'https://app.mi-frontend.com',
    ],

    // Con patrones
    'allowed_origins_patterns' => [
        'https://*.mi-dominio.com',
    ],

    // Cabeceras permitidas en peticiones
    'allowed_headers' => ['*'],

    // Cabeceras expuestas al navegador
    'exposed_headers' => [],

    // Tiempo de caché para preflight (OPTIONS)
    'max_age' => 0,

    // NECESARIO para Sanctum con SPA
    'supports_credentials' => true,
];

Solo en desarrollo — permitir todos los orígenes

'allowed_origins'     => ['*'],
'supports_credentials' => false, // No compatible con '*'

Registrar middleware CORS en bootstrap/app.php

->withMiddleware(function (Middleware $middleware) {
    $middleware->api(prepend: [
        \Illuminate\Http\Middleware\HandleCors::class,
    ]);
})
Si usas Sanctum con SPA, debes tener supports_credentials: true y el origen debe estar explícitamente en allowed_origins (no '*').

Laravel Sanctum — Autenticación de APIs
Sanctum provee autenticación liviana para SPAs, apps móviles y APIs simples basadas en tokens. Es la solución oficial de Laravel para auth de APIs.

Instalación

composer require laravel/sanctum
php artisan vendor:publish --provider="Laravel\Sanctum\SanctumServiceProvider"
php artisan migrate

Modelo User

use Laravel\Sanctum\HasApiTokens;

class User extends Authenticatable
{
    use HasApiTokens, HasFactory, Notifiable;
}

Auth con tokens (APIs móviles)

Route::post('/login', function (Request $request) {
    $request->validate([
        'email'    => 'required|email',
        'password' => 'required',
        'device'   => 'required|string',
    ]);

    $user = User::where('email', $request->email)->first();

    if (!$user || !Hash::check($request->password, $user->password)) {
        return response()->json([
            'message' => 'Credenciales incorrectas'
        ], 401);
    }

    $token = $user->createToken(
        $request->device,
        ['read', 'write']
    )->plainTextToken;

    return response()->json([
        'token' => $token,
        'user'  => $user,
    ]);
});

Route::middleware('auth:sanctum')->group(function () {
    Route::get('/user', fn(Request $r) => $r->user());
    Route::post('/logout', function (Request $r) {
        $r->user()->currentAccessToken()->delete();
        return response()->json(['message' => 'Sesión cerrada']);
    });
});

Migraciones de base de datos
Las migraciones son control de versiones para tu base de datos. Permiten crear, modificar y eliminar tablas de forma reproducible en cualquier entorno.

Comandos esenciales

php artisan make:migration create_productos_table
php artisan make:migration add_precio_to_productos_table --table=productos

php artisan migrate               # Ejecutar pendientes
php artisan migrate:status        # Ver estado de cada migración
php artisan migrate:rollback      # Revertir último batch
php artisan migrate:rollback --step=3
php artisan migrate:reset         # Revertir TODAS
php artisan migrate:refresh       # Reset + migrate
php artisan migrate:fresh         # Drop + migrate (más limpio)
php artisan migrate:fresh --seed  # Con seeders

Eloquent ORM

Modelo completo

class Producto extends Model
{
    use SoftDeletes;

    protected $fillable = ['nombre', 'precio', 'stock', 'categoria_id'];

    protected $casts = [
        'atributos'         => 'array',
        'activo'            => 'boolean',
        'precio'            => 'decimal:2',
        'publicado_en'      => 'datetime',
        'fecha_publicacion' => 'date',
    ];

    protected $hidden = ['deleted_at'];

    public function categoria(): BelongsTo {
        return $this->belongsTo(Categoria::class);
    }

    public function pedidos(): BelongsToMany {
        return $this->belongsToMany(Pedido::class)
                    ->withPivot('cantidad', 'precio_unitario')
                    ->withTimestamps();
    }

    public function scopeActivos(Builder $query): Builder {
        return $query->where('activo', true);
    }

    protected function nombre(): Attribute {
        return Attribute::make(
            get: fn($v) => ucwords($v),
            set: fn($v) => strtolower($v),
        );
    }
}

API Resource Controller

class ProductoController extends Controller { public function index(Request $request): AnonymousResourceCollection { $productos = Producto::query() ->with('categoria') ->when($request->search, fn($q, $s) => $q->where('nombre', 'like', "%$s%") ) ->when($request->categoria, fn($q, $c) => $q->whereHas('categoria', fn($q) => $q->where('slug', $c)) ) ->activos() ->orderBy('nombre') ->paginate(20); return ProductoResource::collection($productos); } public function store(StoreProductoRequest $request): ProductoResource { $producto = Producto::create($request->validated()); return new ProductoResource($producto); } public function show(Producto $producto): ProductoResource { return new ProductoResource($producto->load('categoria')); } }
